# 🤖 뉴스 큐레이션 및 거시경제 보고서 자동화 시스템 최종 보고서

본 보고서는 깃허브 액션(GitHub Actions)과 노션(Notion)을 연동한 지능형 뉴스 큐레이션 및 경제 브리핑 시스템의 구축 결과와 운영 지침을 정리한 문서입니다.

---

## 1. 시스템 개요
본 시스템은 매일 아침 최신 뉴스 데이터를 수집하여 분석하고, 거시경제 관점의 요약 보고서를 생성하여 노션 워크스페이스로 자동 전송하는 '서버리스(Serverless)' 파이프라인입니다.

### 핵심 기능
- **다분야 뉴스 수집**: 경제, 반도체, 세계 경제, 전쟁 등 주요 섹터 뉴스 수집 (NewsAPI 활용)
- **AI 거시경제 브리핑**: Google Gemini(Search Grounding)를 통한 인과관계 분석 및 투자 시나리오 생성
- **노션 자동화**: 수집된 기사 및 생성된 리포트를 노션 데이터베이스/페이지로 자동 업로드
- **클라우드 스케줄링**: 깃허브 액션을 통해 매일 오전 8시(KST) 자동 실행 및 10일 주기 데이터 관리

## 3. 핵심 설정 가이드 (GitHub Secrets)
보안을 위해 모든 API 키는 저장소의 **Settings > Secrets and variables > Actions**에서 관리됩니다.

| Secret Name | 설명 | 주의 사항 |
| :--- | :--- | :--- |
| `NEWS_API_KEY` | NewsAPI.org 발급 키 | - |
| `LLM_API_KEY` | Google Gemini API 키 | **절대 코드에 직접 포함 금지** (유출 시 즉시 차단됨) |
| `NOTION_API_KEY` | 노션 통합(Integration) 토큰 | 노션 페이지에 'JYH' 통합 서비스가 공유되어야 함 |
| `NOTION_DATABASE_ID` | 뉴스 DB ID | 값 끝에 **줄바꿈(\n)**이 포함되지 않도록 주의 |
| `NOTION_BRIEFING_PAGE_ID` | 보고서 페이지 ID | 위와 동일 (줄바꿈 포함 시 400 에러 발생) |

## 4. 주요 장애 해결 기록 (Troubleshooting)

### ① API 키 차단 문제 (Leaked API Key)
- **증상**: `PERMISSION_DENIED` 에러와 함께 Gemini API 호출 실패
- **원인**: 공개 저장소의 `config.py` 파일에 실제 API 키가 포함되어 깃허브 보안 필터에 의해 차단됨
- **해결**: `config.py`에서 실제 값을 제거하고 `os.environ.get().strip()`을 사용하도록 수정하여 보안 강화

### ② 노션 ID 형식 오류 (UUID/Newline)
- **증상**: `database_id should be a valid uuid` 에러 발생
- **원인**: 깃허브 Secrets 등록 시 복사/붙여넣기 과정에서 줄바꿈(Enter)이 포함됨
- **해결**: 코드 내에서 `.strip()` 함수를 사용하여 로드 시 공백과 줄바꿈을 강제로 제거하도록 조치

### ③ 라이브러리 호환성 문제
- **증상**: `ImportError` 또는 `ModuleNotFoundError` 발생
- **원인**: `google-generativeai`(구형)와 `google-genai`(신형) SDK가 혼용되어 발생
- **해결**: `requirements.txt`에 두 버전을 모두 명시하고, 실행 환경(psutil 등) 패키지를 추가하여 호환성 확보

### ④ 노션 속성 제약 (Commas not allowed)
- **증상**: 섹터 업데이트 시 `Invalid select option` 에러 발생
- **원인**: 노션의 '선택(Select)' 속성에는 쉼표(`,`)를 포함할 수 없음
- **해결**: 데이터 전송 전 `.replace(",", " ")` 처리를 통해 쉼표를 공백으로 치환
