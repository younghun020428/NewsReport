# 🤖 뉴스 큐레이션 봇 유지보수 가이드

이 문서는 시스템의 구조, 주요 설정 변경 방법 및 발생 가능한 오류 해결 방법을 정리한 가이드입니다.

---

## 1. 시스템 구조 및 실행 흐름
1. **`run_bot.bat`**: 윈도우 작업 스케줄러가 호출하는 실행 파일입니다.
2. **`scripts/00_main_pipeline.py`**: 전체 프로세스를 제어하는 메인 스크립트입니다.
3. **실행 순서**: 배터리 체크 ➔ 뉴스 수집 ➔ 거시경제 보고서 생성 ➔ 뉴스 필터링 ➔ 노션 개별 뉴스 업로드 ➔ 노션 보고서 업로드 ➔ 10일 지난 데이터 정리.

---

## 2. 주요 설정 변경 방법

### ① 배터리 실행 조건 변경
`scripts/00_main_pipeline.py` 파일의 상단에서 기준값을 수정할 수 있습니다.
- 현재 설정: **70%** (전원 미연결 시 70% 이상일 때만 실행)
- 수정 위치: `if percent >= 70:` 부분을 원하는 숫자로 변경.

### ② 뉴스 검색어 및 API 키 설정
`scripts/config.py` 파일에서 수정합니다.
- `NEWS_QUERY_KEYWORD`: 수집할 뉴스 검색어 (예: "반도체 OR AI")
- `NEWS_API_KEY`: 뉴스 수집 API 키
- `NOTION_API_KEY`: 노션 통합 API 키

## 3. 발생 가능한 오류 및 해결 방법

### ❌ "내부 또는 외부 명령...이 아닙니다" (경로/실행 오류)
- **원인**: 윈도우 환경 변수(`PATH`) 문제 또는 경로 내 공백/한글 처리 오류.
- **해결**: `run_bot.bat`은 현재 `%~dp0`를 사용하여 상대 경로로 동작합니다. 폴더를 통째로 옮기더라도 `run_bot.bat`만 다시 실행하거나 작업 스케줄러를 재등록하면 해결됩니다.

### ❌ 노션 보고서 업로드 실패 (400/404 에러)
- **원인 1 (권한)**: 노션 페이지 상단 [공유] 메뉴에서 **JYH**(통합 서비스)가 추가되어 있는지 확인하세요.
- **원인 2 (데이터베이스 구조)**: 현재 "일일 경제 보고서"는 **데이터베이스** 형식을 사용 중입니다. 만약 이를 일반 **페이지**로 바꾸신다면 `scripts/06_upload_report_to_notion.py`의 `parent` 설정을 `page_id`로 다시 바꿔야 합니다.
- **원인 3 (필드명)**: 데이터베이스의 첫 번째 제목 필드 이름이 **"페이지"**가 아닐 경우 자동으로 찾도록 설계되어 있으나, 구조가 크게 바뀌면 오류가 날 수 있습니다.

### ❌ 자동 실행이 안 됨 (작업 스케줄러)
- **확인**: `Win + R` ➔ `taskschd.msc` 입력 ➔ [NewsCurationMacroBot] 작업이 있는지 확인.
- **재설정**: 관리자 권한 CMD에서 `py setup_task_scheduler.py`를 실행하여 다시 등록하세요.
